# 코트맵 (CourtMap) — 기획서 (v1, MVP)

> 배드민턴 사설 구장 통합 예약 플랫폼. 디자인 AI 입력용. 화면 흐름·톤·기능 정의 중심.
>
> **서비스명**: 코트맵 (CourtMap) — "지도에서 빈 코트를 찾는다"는 핵심 UX를 직설.

---

## 1. 서비스 개요

- **한 줄 요약**: 전국 사설 배드민턴 구장의 빈 코트를 실시간으로 보고 1시간 단위로 즉시 예약하는 **양면 마켓플레이스**.
- **타겟 모델**: 예약 사용자(B2C) + 구장 운영자(B2B) 동등 비중
- **예약 단위**: 코트 1면 × 1시간 (운영자가 날짜별 오버라이드 가능 — 요일·공휴일·특정날짜)
- **핵심 가치 / 차별점**: **"실시간 빈 코트만 한눈에"**
  - 사용자: 전화/카톡 없이 비어있는 코트 즉시 확정 예약
  - 운영자: 공실 자동 노출 + 예약 관리 자동화
- **결제**: MVP는 **무통장입금 (운영자 수동 확인)**. 카드/간편결제 및 분담 결제는 v2.

---

## 2. 타겟 사용자 (페르소나 5종)

### A. 예약 사용자

1. **퇴근 직장인 (30대)** — 평일 저녁 7~10시, 회사·집 근처 즉시 예약. 가까움+빈 시간 우선.
2. **주말 동호회 총무 (30~40대)** — 주말 2~3시간 8명 예약. 정기적으로 같은 시간대 필요. 단체/연속 예약.
3. **레슨 입문자 (20~50대)** — 코치 추천 구장 위주. 강사/레슨 정보 필요.

### B. 구장 운영자

4. **소규모 사설 구장주 (1~4면)** — 카톡/전화 예약 위주. 공실 자동 채우기 원함. PC 익숙도 보통.
5. **대형 체육관 매니저 (5면+)** — 자체 시스템 보유 가능. 외부 노출 채널로 활용. API 연동 필요.

## 3. 핵심 사용자 시나리오 (MVP 6개)

### S1. 퇴근 후 즉시 예약 (페르소나 1)
화요일 18:30 회사 앞 → 지도에서 "지금부터 2시간 안에 빈 코트" 필터 → 도보 10분 거리 발견 → 19:00–20:00 코트 A 선택 → 무통장입금 안내 → 입금 후 운영자 확정 → QR 입장코드. **목표 소요: 1분 이내 (예약 신청까지).**

### S2. 주말 단체 정기 예약 (페르소나 2)
일요일 13~15시 매주 반복 → 캘린더에서 "이 요일·시간 반복" → 4주치 한 번에 예약 신청 → 입금 후 일괄 확정 → 동호회 단톡방에 공유. **핵심: 다회 예약 + 공유 링크.**

### S3. 공휴일 2시간 묶음 예약 (페르소나 1·2)
12/25 검색 → 평소 1시간 단위인 구장이 "이 날은 2시간 단위" 배지 표시 → 14:00–16:00 슬롯만 선택 가능. **핵심: 슬롯 단위가 다른 날 UI 명확성.**

### S4. 운영자 공실 자동 채우기 (페르소나 4)
운영자 모바일 어드민 → "오늘 19~21시 코트 B 비어있음" 알림 → "긴급 할인 30%" 토글 → 메인 "지금 핫딜" 영역 노출 → 30분 안에 예약 들어옴. **핵심: 다이나믹 프라이싱.**

### S5. 대형 체육관 외부 채널 활용 (페르소나 5)
자체 시스템 운영 매니저 → 우리 플랫폼은 "남는 시간만 노출" 모드 → iCal/Webhook으로 양방향 동기화. **핵심: API 연동.**

### S6. 노쇼 페널티 (전체)
예약 후 미입장 → 운영자가 입장 체크 안 한 시점 기준 자동 노쇼 처리(또는 운영자 수동 신고) → 사용자 신뢰점수 차감 → 일정 점수 이하 시 일정 기간 예약 제한. **MVP는 보증금 없이 점수+제한만.**

> **v2 이관**: S7 분담 결제 (카드 결제 도입 시 함께)

## 5. 주요 기능 정의

### 예약 슬롯 규칙 (S3 핵심)
- **기본**: 1시간 단위
- **운영자 오버라이드**: 요일별 / 공휴일 / 특정 날짜에 2H·3H 단위 강제 가능
- **공휴일 자동 동기화**: 한국 공휴일 API 연동
- **UI 표시**: 캘린더에서 단위가 다른 날은 배지로 시각화

### 결제 (MVP = 무통장입금)
1. 사용자 예약 신청 → "입금 대기" 상태
2. 사용자 입금 → 운영자가 O12에서 입금자명 매칭 → "확정" 처리
3. 입금기한 초과 시 자동 취소
4. 환불 정책: 24h 전 100% / 1h 전 50% / 이후 0% (운영자별 조정 가능)

### 다이나믹 프라이싱 (S4)
- 운영자가 임박 공실에 즉시 할인 적용
- 메인 핫딜 영역에 노출
- 알림 — 단골 사용자에게 푸시/카톡

### 노쇼 페널티 (S6)
- 자동: 운영자가 입장 체크 안 한 시점 기준 노쇼 처리
- 수동: 운영자 신고 가능
- 결과: 신뢰점수 차감 → 일정 이하 시 예약 제한 N일

### 외부 시스템 연동 (S5)
- iCal export (구장 → 외부)
- Webhook (예약 확정 시 외부 시스템에 통보)
- v2: 양방향 API

### 알림 시스템
- 채널: 푸시 + 카카오 알림톡
- 종류: 예약 리마인더 / 노쇼 경고 / 핫딜 / 단골 빈자리 / 친구 정산 요청(v2)

### 쿠폰·멤버십
- 쿠폰: 운영자가 발행, 사용자는 결제 시점 적용
- 멤버십: 월 N시간 정액 (정기 사용자용 매출 안정화)

### 장비 대여
- 예약 시 옵션 추가 결제 (라켓·셔틀콕)
- 운영자가 옵션·가격 등록

### 레슨 (페르소나 3)
- 구장 상세에 강사 탭
- 코트 예약과 별도 흐름 (강사·시간 선택)

## 6. 톤앤무드

- **방향**: **B. 깔끔한 부킹앱** — 화이트 베이스, 미니멀, 정보 밀도 높음
- **레퍼런스**: **야놀자 톤** (검색·필터·예약 흐름 명확, 모바일 최적화)
- **메인 컬러**: **코발트블루 (#1E50FF 계열) + 화이트**
- **타이포**: 한글은 Pretendard / Noto Sans KR 계열, 정보 위주 화면이라 가독성 우선
- **느낌 키워드**: *신뢰감, 빠름, 명확함, 부담 없음*
- **이미지 스타일**: 일러스트 최소화, 실제 구장 사진 위주, 깔끔한 정렬
- **모바일 우선**: 사용자 측은 모바일 First, 운영자 측은 PC + 모바일 둘 다

## 8. 디자인 AI에 전달할 우선순위

디자인 AI에 부분 발주 시 권장 순서:
1. **U1 홈** + **U2 검색결과/지도** (가장 임팩트 큰 진입 화면)
2. **U3 구장 상세** + **U4 예약 시간 선택** (S3 캘린더 UI가 핵심)
3. **U5 입금 안내** + **U6 예약 완료** (무통장입금 흐름의 신뢰감)
4. **O1 운영자 대시보드** + **O3 슬롯 규칙 편집** (운영자 측 핵심)
5. 나머지는 위 패턴을 확장

## 9. 운영자/관리자 화면 상세 명세

> 사용자측 프론트(zip)는 받았지만 운영자/관리자 측은 별도 디자인 작업이 필요. 디자인 AI에 넘길 수 있도록 화면별 핵심 요소를 정리.
>
> **레이아웃**: PC 우선 (좌측 사이드바 네비 + 우측 메인). 모바일 대응은 운영자 핵심 동작(O5 예약 관리·O12 입금 확인)에 집중.
> **톤**: 사용자측과 동일 (코발트블루 #1E50FF + 화이트, 야놀자 톤). 정보 밀도는 더 높게.

### 운영자 측 (O1~O12)

**O1. 운영자 대시보드**
- 상단 KPI 카드 4개: 오늘 예약 수(확정/대기), 공실률, 오늘 매출, 이번 주 누적
- 임박 공실 알림 영역: 다음 N시간 빈 코트 리스트 → 1클릭 핫딜 발행
- 오늘 일정 타임라인 (시간축, 코트별 가로 줄)
- Quick actions: "입금 확인", "오늘 노쇼 처리", "슬롯 규칙 수정"

**O2. 구장/코트 관리**
- 구장 기본정보: 이름, 주소, 좌표(지도 핀), 연락처, 사진 다중 업로드
- 코트 목록: A/B/C/D 행, 가격 오버라이드 컬럼, 추가/삭제 버튼
- 시설 태그 체크박스 (주차/샤워/라켓대여/에어컨/락커/정수기 등)
- 입금 계좌: 은행·계좌번호·예금주
- 환불 정책: 24h%, 1h%, 1h이내% (기본 100/50/0)
- 운영시간: 요일별 open~close, 휴무 토글
- 임시 휴무 추가 (캘린더 클릭)

**O3. 슬롯 규칙 편집**
- 기본 슬롯 단위 라디오 (1H/2H/3H)
- 요일별 오버라이드 (월~일 각 1/2/3H 드롭다운)
- 공휴일 자동 적용 토글 + 공휴일용 슬롯 단위
- 특정 날짜 추가: 캘린더에서 클릭 → 슬롯 단위 지정
- 한국 공휴일 자동 동기화 상태 표시 (마지막 동기화 일자)

**O4. 다이나믹 프라이싱**
- 활성 핫딜 카드 리스트 (남은시간·할인율·코트 표시, 즉시 취소)
- 수동 발행 폼: 날짜·시간·코트(또는 전체)·할인율·만료시각
- 자동 룰 섹션: 룰 생성/수정/일시정지
  - "시작 N시간 전 빈 코트 → 자동 X% 할인"
  - 적용 요일 (체크박스 7개), 적용 시간대 (from~to)

**O5. 예약 관리**
- 상단 탭: 오늘 / 내일 / 이번 주
- 필터: 상태(확정/대기/완료/노쇼/취소), 코트, 검색
- 리스트: 시간 / 사용자(이름·전화·신뢰점수) / 코트 / 금액 / 상태 / 액션
- 액션 버튼: **입장 체크**, **노쇼 신고**, 취소, 메모
- 캘린더 뷰 토글 (시간 × 코트 그리드)

**O6. 정산/매출**
- 매출 차트 (일/주/월)
- 코트별·시간대별 매출 분포 (히트맵)
- 정산 내역 테이블 (월별, 입금/환불/순매출)
- CSV/엑셀 다운로드

**O7. API 연동 설정 (S5)**
- API 토큰 발급: 이름·스코프 선택 → 토큰 1회 노출 + 복사
- 토큰 목록: 마지막 사용 시각, 폐기 버튼
- iCal 피드 URL (구장 캘린더 export, 복사 버튼)
- Webhook 등록: 이벤트 종류·URL·시크릿
- Webhook 호출 로그 (성공/실패, 재시도)

**O8. 쿠폰·멤버십 발행**
- 쿠폰 탭: 신규 쿠폰 생성(이름·할인종류·값·최소금액·유효기간·총발행수), 발행 쿠폰 리스트(사용 현황)
- 멤버십 탭: 상품 등록(이름·가격·시간/월·유효기간), 상품 목록, 가입자 수

**O9. 장비 대여 옵션 관리**
- 장비 카드 (라켓/셔틀콕/기타) 추가
- 가격, 설명, 활성/비활성 토글
- 정렬 순서 드래그

**O10. 1:1 문의 응대**
- 문의 리스트 (사용자별, 답변 대기 우선)
- 채팅 패널 (좌:리스트 / 우:대화)
- 자주쓰는 답변 단축키 (선택)

**O11. 강사·레슨 관리**
- 강사 등록: 이름, 경력, 프로필 사진, 회당 가격, 회당 시간
- 강사 목록 (활성/비활성)
- 레슨 예약 리스트 (강사별 필터)

**O12. 입금 확인** ← MVP에서 가장 중요한 운영 동작
- 입금 대기 예약 리스트 (입금기한 임박순)
- 각 행: 예약자명·입금자명·금액·예약 정보·입금기한 카운트다운
- 자동 매칭 후보 표시 (입금자명+금액 일치 → 초록 점)
- 1클릭 **확정** / 보류 / 취소 버튼
- 모바일 우선 디자인 (운영자가 수시로 핸드폰으로 확인)

### 관리자 측 (A1~A4)

**A1. 어드민 대시보드**
- 전체 KPI: 누적 거래액, 활성 구장, 신규 가입(주간), 활성 사용자
- 신규 구장 신청 알림 카운트
- CS 대기 건수
- 시스템 헬스 (API 응답·DB·웹훅 실패율)

**A2. 구장 승인**
- pending 구장 리스트
- 상세 검토 페이지: 정보·사진·운영자 정보
- 승인 / 반려(사유 입력)

**A3. 사용자/운영자 관리**
- 사용자 검색 (이름·전화·이메일)
- 상세: 신뢰점수·예약 이력·노쇼 로그·환불계좌
- 액션: 점수 수동 조정, 일시 정지, 영구 정지, 강제 탈퇴

**A4. 신고/문의 처리**
- 사용자 신고 리스트 (구장 신고, 운영자 신고, 사용자 간 신고)

- 상태 토글 (대기/처리중/완료)

## 10. 신뢰점수 정책 (확정)

| 항목 | 점수 변동 |
|---|---|
| 시작 점수 | 100 |
| 자동 노쇼 (시작 후 10분 입장 체크 ❌) | −15 |
| 수동 노쇼 (운영자 신고, 관리자 확정) | −15 |
| 정상 이용 완료 | +1 (월 최대 +5) |

**제한 단계**

| 점수 구간 | 제재 |
|---|---|
| 80~100 | 제한 없음 |
| 60~79 | 다음 예약 시 입금기한 12시간으로 단축 |
| 40~59 | 7일 신규 예약 제한 |
| 0~39 | 30일 신규 예약 제한 |

해제 시점은 `users.restricted_until` 으로 관리.

## 11. 다이나믹 프라이싱 정책

- **수동 모드**: 운영자가 핫딜 즉시 발행 (O4 화면)
- **자동 모드**: 운영자가 룰 등록 → 시스템이 조건 만족 시 자동 발행
  - 트리거: "시작 N시간 전까지 비어있으면" (N=1~6)
  - 적용 요일·시간대 제한
  - 룰 활성화 토글로 ON/OFF
- 두 모드 동시 사용 가능

## 12. 인증 정책

- **소셜로그인 1순위**: 카카오 (Kakao OAuth)
- 이메일+비번 가입도 동시 지원
- **회원가입 필수 입력**: 이메일·전화번호·이름·환불계좌(은행·계좌번호·예금주)
- 환불계좌가 필수인 이유: 무통장입금 환불을 자동 송금 또는 운영자 수동 송금으로 처리하기 위함
